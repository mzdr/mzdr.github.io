Introduction

You're viewing the demonstration site of Orientation.js, a tiny JavaScript library that helps you and your visitors keep track of knowing where they are on your website.

It's very simple to set up, no matter if it's a new website or an existing one. If it's a gallery, a portfolio, a forum or a blog — whatever it is, it will help your visitors getting along with your website!

Just watch the navigation on top of this site while scrolling/swiping to grasp the idea and check out the usage section to get to know how to do it!

Downloads

Orientation is open source and always will be! You can download stable releases, the latest development state or the source code itself over at the official GitHub repository.

Usage

Before we dig right into it you should load the almighty magic. Place the following code somewhere between the head or body tag.  —  Don't forget to adjust the paths!

<script src="path/to/orientation.min.js"></script>
<script>Orientation.init({ options });</script>

Now as you're ready to go you should know that Orientation is super flexible! But where to start? The most basic way to use it is by defining just a target. In that case Orientation is going to watch the targets position when the window (by default) is getting scrolled.

So for example you have a <div> element where you want to show the progress of the whole website. All you need to do is to set it up with the html selector as the target.

<div data-orjs-target="html"></div>

After the initializing the <div> will get additional data attributes that represent the collected data.

<div data-orjs-target="html" data-orjs-status="is-there" data-orjs-progress="23"></div>

Starting with data-orjs-status which is just a simple status with three states given: is-upcoming, is-there and is-gone. They just tell roughly the position relative to the container (which is by default the window).

On the other hand there is data-orjs-progress which will give you a percentage between 0 – 100 where 0 means that the target element is not visible and at 100 it's fully visible.

There's another data attribute called data-orjs-style. It's perfect if you want to use the scrolling progress as a value for a specific CSS style property which needs to be written in camelCase!

So let's say you add this attribute and want it to mirror the progress by the width of the element.

<div class="progress-bar" data-orjs-target="html" data-orjs-style="width"></div>
