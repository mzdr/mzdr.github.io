Every web project needs to have a beginning. This is it!

It first started as a simple collection of scripts, markup snippets and css definitions. I always used to need them when I set up a new project. From time to time things have been improved, deleted and refactored. I realized that most of the time I needed to look through the latest project in order to start a new one.

That's why I began to wonder, why not setup your own boilerplate?

So this is how a new web project looks like when you set it up with the daiippo boilerplate:

├── public (production files)
│   ├── assets
│   │   ├── css
│   │   │   ├── vendor (3rd party styling)
│   │   │   └── …
│   │   ├── fonts
│   │   │   ├── fontname (each font family goes into a separate folder)
│   │   │   └── …
│   │   ├── images
│   │   │   └── …
│   │   └── js
│   │       ├── vendor (3rd party javascript)
│   │       └── …
├── index.html
├── …
└── src (working files)
    ├── scripts (custom javascript applications)
    └── styles
        ├── main.sass
        ├── meta (everything that won't be rendered as css)
        │   ├── _config.sass
        │   └── …
        ├── print.sass
        └── shared (styling that's shared among multiple elements)
            ├── _elements.sass
            ├── _layout.sass
            ├── _typography.sass
            └── ui (user interface elements)
                ├── _button.sass
                ├── _form.sass
                ├── _list.sass
                └── _navigation.sass
